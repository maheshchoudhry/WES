# WES OS v0.2.0-alpha — Release Notes

**Release date:** 2026-07-14
**Status:** AI Company Work Management Platform (alpha) · Runtime verified
**Previous release:** v0.1.0-alpha

## Overview

v0.2.0-alpha turns WES OS from a foundation platform into an **AI software
company that can manage software development**. It adds a first-class AI
organization (Sprint 06) and an operational work-execution engine (Sprint 07):
AI employees now receive, own, track, and complete work across projects, sprints,
and a Kanban board — with no LLM/AI execution, purely the operational engine.

Start it with:

```bash
./scripts/bootstrap.sh && ./scripts/dev.sh
```

## Major Features

- **AI Company Core** — AI employees as first-class entities with departments,
  roles, capabilities, responsibilities, KPIs, and a reporting hierarchy;
  dedicated AI workspaces and organization chart.
- **Work Management** — Projects, Milestones, Sprints, Work Items (tasks),
  Assignments, Dependencies, and an Activity Timeline.
- **Kanban Task Board** — seven ordered columns with per-card status moves.
- **Assignment Engine** — routes work to AI employees, auto-transitions status,
  and records the timeline.
- **Dashboards** — Founder and AI dashboards extended with organization health
  and work analytics (velocity, blocked, workload, department load).

## Technical Highlights

- Layered FastAPI + SQLAlchemy backend; React + TypeScript + Vite frontend.
- Two new migrations (`0003`, `0004`) — 16 new tables — with the portable UUID
  type; self-initializing backend (auto-migrate + seed).
- RBAC extended (`ai:*`, `work:*`) on the reusable permission middleware.
- Verified quality: **132 backend** + **25 frontend** tests, ruff/black/prettier,
  and a passing runtime health suite (10/10).

## Database Changes

- `0003_ai_company_core`: ai_departments, ai_roles, ai_capabilities, ai_employees,
  ai_responsibilities, ai_kpis, ai_employee_capabilities.
- `0004_work_management`: projects, milestones, project_sprints, work_items,
  assignments, work_dependencies, activity_log, comments, attachments_metadata.

## API Changes

New endpoints: `/ai-employees` (CRUD), `/ai-roles`, `/ai-departments`,
`/ai-org/{chart,departments,summary}`, `/projects` (CRUD + nested),
`/sprints`, `/milestones`, `/tasks` (CRUD + assign/dependencies/comments),
`/assignments`, `/activity`, `/work/{kanban,founder-summary,ai-summary}`.

## Dashboard Changes

- **Founder Dashboard** — AI Organization summary + Work Management summary
  (projects, tasks, blocked, velocity).
- **AI Company Dashboard** — organization stats + Work summary
  (assigned/in-progress/completed, department load).

## Known Limitations

- Kanban move is a status selector (DnD-ready), not native drag-and-drop.
- Attachments are metadata-only; comments/dependencies have minimal UI.
- Work RBAC enforces `work:read`/`work:write`; AI-role approval semantics live in
  the workflow/seed narrative.
- Analytics load full tables in memory (fine at current scale).
- Docker/PostgreSQL path is config-validated only in local dev.

## Sprint 08 Goals

- True drag-and-drop Kanban (dnd-kit) on the existing status-move foundation.
- Sprint planning & burndown charts.
- Notifications on assignment / blocked / status change.
- Department-scoped work RBAC and a per-assignee "My Work" view.
- File attachments (upload + storage).
