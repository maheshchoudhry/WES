# WES OS v0.1.0-alpha — Release Notes

**Release date:** 2026-07-13
**Status:** First operational release (alpha) · Runtime verified

## Overview

WES OS (WORLD Engineering Studio Operating System) reaches its first operational
milestone. Five foundation sprints deliver a working, authenticated executive
operating system with a professional one-command developer experience. This is an
**alpha** foundation release: the core platform is complete and verified, business
modules beyond the Company Engine arrive in later releases.

Start it with:

```bash
./scripts/bootstrap.sh && ./scripts/dev.sh
```

## Major Features

- **Company Engine** — manage Company, Departments, and Employees with validation,
  business rules, and full REST CRUD.
- **Founder Dashboard** — an executive view of the company on live data:
  department cards, employee workspace, org hierarchy, company health, recent activity.
- **Authentication & RBAC** — secure login (bcrypt + JWT), refresh tokens, logout
  invalidation, and five roles enforced on every API.
- **One-command developer platform** — start, stop, reset, health-check, bootstrap,
  test, lint, and format the entire stack with single commands.

## Technical Highlights

- Layered backend (FastAPI · SQLAlchemy 2 · Alembic · Pydantic v2) with a standard
  response/error envelope and self-initialization (auto-migrate + seed on startup).
- React 18 + TypeScript + Vite frontend with session management, protected routes,
  transparent token refresh, and 401/403 handling.
- RBAC permission matrix with reusable `require_permission` middleware.
- Portable database layer (PostgreSQL prod / SQLite dev & tests) via a cross-dialect
  UUID type; migrations `0001_initial` + `0002_auth_fields`.
- Verified quality: **77 backend tests**, **20 frontend tests**, ruff/black/prettier
  formatting, and a runtime health suite (10/10 checks passing).

## Known Issues

- Tokens are stored client-side (bearer), not httpOnly cookies.
- Development JWT secret and seeded passwords are placeholders — rotate before deploy.
- Director / Department-Head write permissions are company-wide (not yet scoped to a
  department).
- The activity feed is derived from entity timestamps (no audit log yet).
- The Docker/PostgreSQL path is configured but validated only by config in local dev.
- React Router v7 future-flag warnings appear in test output (cosmetic).

## Next Release Goals
