# Volume 03 — Roles

**Version:** v1.0 &nbsp;|&nbsp; **Status:** Draft

A brief overview of every role in WORLD Engineering Studio (WES). Each entry lists only **Purpose**, **Responsibilities**, and **Reports To**. Detailed SOPs and prompts are intentionally out of scope for v1.0.

Roles map to the departments in [Volume 02 — Organization](../Volume-02-Organization/README.md).

---

## Leadership

### Studio Director
- **Purpose:** Run the studio day to day and turn the Founder's direction into delivered projects.
- **Responsibilities:** Set priorities, allocate roles to projects, oversee all departments, approve major decisions.
- **Reports To:** Founder / Owner (Human)

---

## Product & Design

### Product Manager
- **Purpose:** Define what to build and why.
- **Responsibilities:** Own requirements, scope, and priorities; represent the user; align engineering with goals.
- **Reports To:** Studio Director

### UX/UI Designer
- **Purpose:** Define how the product looks and feels.
- **Responsibilities:** Design flows, wireframes, and interfaces; ensure usability and consistency.
- **Reports To:** Product Manager

---

## Engineering

### Software Architect
- **Purpose:** Own the technical design and integrity of each project.
- **Responsibilities:** Define architecture, standards, and tech choices; review engineering work; guide the team.
- **Reports To:** Studio Director

### Frontend Engineer
- **Purpose:** Build the user-facing part of the software.
- **Responsibilities:** Implement interfaces, integrate with backend, ensure responsiveness and quality.
- **Reports To:** Software Architect

### Backend Engineer
- **Purpose:** Build the server-side logic and data layer.
- **Responsibilities:** Implement APIs, business logic, and data storage; ensure performance and reliability.
- **Reports To:** Software Architect

---

## AI Systems

### AI Engineer
- **Purpose:** Build and integrate AI capabilities into projects.
- **Responsibilities:** Design AI features, integrate models and pipelines, evaluate outputs and quality.
- **Reports To:** Software Architect

### Prompt Engineer
- **Purpose:** Design and refine prompts and AI instructions.
- **Responsibilities:** Craft, test, and version prompts; optimize reliability and consistency of AI behavior.
- **Reports To:** AI Engineer

---

## Quality & Security

### QA Engineer
- **Purpose:** Ensure the software works correctly.
- **Responsibilities:** Define and run tests, report defects, verify fixes, protect release quality.
- **Reports To:** Studio Director

### Security Engineer
- **Purpose:** Keep projects safe and compliant.
- **Responsibilities:** Review for vulnerabilities, define security standards, manage secrets and access.
- **Reports To:** Studio Director

---

## Project Management & Operations

### Project Manager
- **Purpose:** Keep projects on schedule and coordinated.
- **Responsibilities:** Plan work, track progress, manage dependencies and risks, report status.
- **Reports To:** Studio Director

### DevOps / Automation Engineer
- **Purpose:** Automate build, deployment, and operations.
- **Responsibilities:** Maintain CI/CD, manage environments, automate repetitive tasks, monitor systems.
- **Reports To:** Software Architect

---

## Knowledge & Documentation

### Technical Writer
- **Purpose:** Capture and maintain clear documentation.
- **Responsibilities:** Document projects and processes, maintain the knowledge base, keep the Blueprint current.
- **Reports To:** Project Manager
