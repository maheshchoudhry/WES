# Project Lifecycle

**Version:** v1.0 &nbsp;|&nbsp; **Status:** Active &nbsp;|&nbsp; **Owner:** WES

The complete lifecycle every WES project moves through.

```
Request → Approval → Initialization → Planning → Design → Development
   → Testing → Deployment → Maintenance → Closure
```

| Stage | Definition |
|-------|------------|
| **Project Request** | A project need is proposed with goal and rationale. |
| **Project Approval** | The Studio Director (Founder if strategic) approves the project to proceed. |
| **Project Initialization** | The project space, profile, and roles are set up. |
| **Planning** | Scope, milestones, tasks, timeline, and risks are defined. |
| **Design** | Product and technical design are produced. |
| **Development** | The software is built per the engineering system ([Volume 04](../../Blueprint/Volume-04-Engineering-System/README.md)). |
| **Testing** | Quality and security verification against acceptance criteria. |
| **Deployment** | The release is delivered per the release process. |
| **Maintenance** | Ongoing fixes, support, and improvements. |
| **Project Closure** | Milestones and Definition of Done confirmed; lessons captured; docs archived. |

Stages align with the [Information Flow](../../Company/Communication-System/Information-Flow.md). Blocked work follows the [Escalation Flow](../../Company/Communication-System/Escalation-Flow.md).
