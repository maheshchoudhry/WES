# Sprint Framework

**Version:** v1.0 &nbsp;|&nbsp; **Status:** Active &nbsp;|&nbsp; **Owner:** WES

Work runs in short, fixed iterations (sprints). Brief descriptions of each ceremony:

| Ceremony | Description |
|----------|-------------|
| **Sprint Planning** | Set the sprint goal and commit to a task set from the backlog. |
| **Sprint Execution** | The team works the committed tasks; status is tracked daily. |
| **Sprint Review** | Completed work is reviewed against the sprint goal and acceptance criteria. |
| **Sprint Retrospective** | The team captures what worked and what to improve; lessons feed the [knowledge base](../../Blueprint/Volume-09-Knowledge-Management/README.md). |

The Project Manager facilitates the sprint; incomplete work is re-planned into the next sprint.
