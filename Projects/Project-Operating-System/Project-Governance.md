# Project Governance

**Version:** v1.0 &nbsp;|&nbsp; **Status:** Active &nbsp;|&nbsp; **Owner:** WES

How projects are governed at WES.

| Area | Rule |
|------|------|
| **Ownership** | Every project has one accountable owner (the Project Manager, under the Studio Director). |
| **Accountability** | Each role is accountable for its tasks and deliverables to its reporting role. |
| **Approval Authority** | Approvals follow the [Approval Flow](../../Company/Communication-System/Approval-Flow.md); strategic or high-risk decisions require human approval. |
| **Change Control** | Scope changes are requested explicitly, assessed for impact, and approved before the plan is updated ([Volume 07](../../Blueprint/Volume-07-Project-Management/README.md)). |
| **Issue Management** | Issues are logged, prioritized, owned, and tracked to resolution. |
| **Risk Management** | Risks are identified early, rated by likelihood and impact, assigned an owner and mitigation, and revisited at each milestone. |
