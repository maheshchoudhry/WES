# Project Operating System (POS)

**Version:** v1.0 &nbsp;|&nbsp; **Status:** Active &nbsp;|&nbsp; **Owner:** WES

The official framework for how every WES software project is managed, from initiation to completion. It applies to all future projects, including [Project-001 (WORLD)](../Project-001-WORLD/README.md). This is an operational framework only — no software, automation, or prompts. It extends [Volume 07 — Project Management](../../Blueprint/Volume-07-Project-Management/README.md) and uses the [Employee Operating System](../../Company/Employee-Operating-System/README.md) and [Communication System](../../Company/Communication-System/README.md).

## Project Classification

| Category | Description |
|----------|-------------|
| **Internal Projects** | Studio tooling, Blueprint, or company infrastructure. |
| **Client Projects** | Software delivered for an external client. |
| **Research Projects** | Time-boxed exploration to reduce uncertainty. |
| **Experimental Projects** | Prototypes and proofs of concept, low commitment. |
| **Maintenance Projects** | Ongoing support, fixes, and improvements to existing software. |

_Project-001 (WORLD) is classified as an **Internal Project**._

## Standard Project Structure

Every project defines the following at initiation:

| Element | Description |
|---------|-------------|
| **Project Profile** | ID, name, category, owner, status, version. |
| **Scope** | What is included and explicitly excluded. |
| **Objectives** | The measurable goals of the project. |
| **Deliverables** | The concrete outputs to be produced. |
| **Timeline** | Target start, key dates, and end. |
| **Milestones** | Verifiable checkpoints with completion criteria. |
| **Risks** | Known risks with likelihood, impact, and mitigation. |
| **Dependencies** | Internal and external dependencies. |
