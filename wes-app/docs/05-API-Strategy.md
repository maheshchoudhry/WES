# API Strategy

**Version:** v1.0 &nbsp;|&nbsp; **Status:** Frozen &nbsp;|&nbsp; **Owner:** WES Engineering

The API contract strategy. **No endpoints are implemented** in Sprint 01.

## Versioning

- All endpoints live under a version prefix: `/api/v1`.
- Breaking changes introduce a new version (`/api/v2`); old versions remain until deprecated.

## Naming Standards

- Resources are **plural nouns** in `kebab-case`: `/api/v1/employees`, `/api/v1/projects`.
- Sub-resources nest: `/api/v1/projects/{id}/tasks`.
- Actions use HTTP verbs (`GET`, `POST`, `PATCH`, `DELETE`), not verbs in the path.
- JSON payloads use `camelCase` (mapped from backend `snake_case`).

## Error Handling

Consistent error envelope with appropriate HTTP status codes:

```json
{
  "error": {
    "code": "RESOURCE_NOT_FOUND",
    "message": "Employee not found",
    "details": {}
  }
}
```

- `4xx` for client errors, `5xx` for server errors.
- Validation errors return `422` with field-level details.
- Every error carries a stable machine-readable `code`.

## Authentication Strategy

- **OAuth2 password flow** issuing **JWT** access tokens; `Authorization: Bearer <token>`.
- Protected routes require a valid token; authorization is enforced by role/permission (RBAC).
- Tokens are short-lived; refresh strategy defined in the Authentication sprint.
- High-risk actions honor the [Human Approval Model](../../Blueprint/Volume-05-AI-System/README.md).

## Conventions

- List endpoints support pagination, filtering, and sorting via query parameters.
- Responses are consistent: a `data` object/array and, for lists, pagination metadata.
- OpenAPI schema is auto-generated by FastAPI.
