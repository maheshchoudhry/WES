# API Strategy

**Version:** v1.1 &nbsp;|&nbsp; **Status:** Frozen &nbsp;|&nbsp; **Owner:** WES Engineering

The API contract strategy. Endpoints for the Company Engine (companies, departments, employees) are implemented in Sprint 02; later modules follow the same rules.

## Versioning

- All endpoints live under a version prefix: `/api/v1`.
- Breaking changes introduce a new version (`/api/v2`); old versions remain until deprecated.

## Naming Standards

- Resources are **plural nouns** in `kebab-case`: `/api/v1/employees`, `/api/v1/projects`.
- Sub-resources nest: `/api/v1/projects/{id}/tasks`.
- Actions use HTTP verbs (`GET`, `POST`, `PATCH`, `DELETE`), not verbs in the path.
- JSON payloads use `camelCase` (mapped from backend `snake_case` via Pydantic alias generator).

## Response Shapes

- **List** endpoints return an envelope: `{ "data": [ ... ], "pagination": { "page", "pageSize", "total" } }`.
- **Single-resource** endpoints (`GET/POST/PATCH /{id}`) return the resource object directly.
- **DELETE** returns `204 No Content`.

## Error Handling

Consistent error envelope with appropriate HTTP status codes:

```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "Employee not found",
    "details": {}
  }
}
```

- `4xx` for client errors, `5xx` for server errors.
- Validation errors return `422` with code `VALIDATION_ERROR` and field-level details.
- Duplicate unique keys return `409` with code `CONFLICT`.
- Missing resources return `404` with code `NOT_FOUND`.
- Every error carries a stable machine-readable `code`.

## Authentication Strategy

- **OAuth2 password flow** issuing **JWT** access tokens; `Authorization: Bearer <token>`.
- Protected routes require a valid token; authorization is enforced by role/permission (RBAC).
- Tokens are short-lived; refresh strategy defined in the Authentication sprint (Sprint 02 of the roadmap; the Company Engine was prioritized first per the Sprint 02 brief).
- High-risk actions honor the [Human Approval Model](../../Blueprint/Volume-05-AI-System/README.md).

## Conventions

- List endpoints support pagination via `page` and `pageSize` query params; employees also support a `departmentId` filter.
- OpenAPI schema is auto-generated by FastAPI and served at `/docs`.
