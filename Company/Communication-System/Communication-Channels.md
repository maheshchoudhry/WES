# Communication Channels

**Version:** v1.0 &nbsp;|&nbsp; **Status:** Active &nbsp;|&nbsp; **Owner:** WES

The official communication channels used inside WES. Each channel is a defined space for a type of communication; all are backed by written, traceable records in the repository.

| Channel | Purpose | Primary Participants |
|---------|---------|----------------------|
| **Executive Communication** | Direction, priorities, approvals, and strategic decisions. | Founder / Owner, Studio Director |
| **Engineering Communication** | Architecture, implementation, and technical decisions. | Software Architect, Engineers, AI Systems |
| **Project Communication** | Planning, status, dependencies, and risks. | Project Manager, all assigned roles |
| **Documentation Communication** | Knowledge, decisions, and documentation updates. | Technical Writer, all roles |
| **Review Communication** | Code review, quality, and security feedback. | QA Engineer, Security Engineer, Software Architect |

## Principles

- Each channel has a clear purpose; communication goes to the right channel.
- All channels use written, traceable records (issues, pull requests, documents).
- Decisions made in any channel are recorded per [Volume 09](../../Blueprint/Volume-09-Knowledge-Management/README.md).
