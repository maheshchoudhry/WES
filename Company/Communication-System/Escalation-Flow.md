# Escalation Flow

**Version:** v1.0 &nbsp;|&nbsp; **Status:** Active &nbsp;|&nbsp; **Owner:** WES

How blocked work is escalated in WES. An employee in the **Blocked** state ([Employee States](../Employee-Operating-System/Employee-States.md)) raises the issue promptly with the context needed to resolve it.

## General Path

```
Blocked Employee → Reporting Role → Studio Director → Founder / Owner
```

## Escalation Types

| Type | Trigger | Path |
|------|---------|------|
| **Technical Escalation** | Architecture, implementation, or tooling blocker. | Engineer → Software Architect → Studio Director |
| **Business Escalation** | Scope, priority, or strategic conflict. | Employee → Product Manager / Studio Director → Founder |
| **Project Escalation** | Schedule, dependency, or risk blocker. | Employee → Project Manager → Studio Director |

## Rules

- Escalate early; do not stay blocked silently.
- Escalate with context: what is blocked, why, and what is needed.
- High-risk or strategic blockers rise to the Founder / Owner.
