# Company Memory System (CMS)

**Version:** v1.0 &nbsp;|&nbsp; **Status:** Active &nbsp;|&nbsp; **Owner:** WES

The official system for how WORLD Engineering Studio (WES) captures, stores, manages, and reuses organizational knowledge across every project. This is an operational framework only — no software, automation, or prompts. It extends [Volume 09 — Knowledge Management](../../Blueprint/Volume-09-Knowledge-Management/README.md) and governs the content stored in the [Knowledge Base](../../KnowledgeBase/README.md).

## Components

| Component | Purpose |
|-----------|---------|
| [Decision Management](./Decision-Management.md) | How important decisions are recorded. |
| [Lessons Learned](./Lessons-Learned.md) | How knowledge from completed work is captured. |
| [Best Practices](./Best-Practices.md) | How reusable best practices are maintained. |
| [Knowledge Lifecycle](./Knowledge-Lifecycle.md) | How knowledge is created through to archived. |
| [Knowledge Governance](./Knowledge-Governance.md) | Ownership, review, approval, and versioning. |

## Company Knowledge Architecture

Organizational knowledge is organized into six categories:

| Category | Purpose |
|----------|---------|
| **Organizational Knowledge** | Company structure, roles, and policies — how WES is organized. |
| **Engineering Knowledge** | Architecture, standards, and technical patterns — how WES builds. |
| **Project Knowledge** | Project profiles, plans, and outcomes — what WES has delivered. |
| **Technical Knowledge** | Technologies, tools, and references — what WES uses. |
| **Business Knowledge** | Goals, priorities, and context — why WES does what it does. |
| **Operational Knowledge** | Processes and operating systems (EOS, POS, CMS) — how WES runs. |

Content for these categories lives in the [Knowledge Base](../../KnowledgeBase/README.md); the CMS defines how it is managed.
