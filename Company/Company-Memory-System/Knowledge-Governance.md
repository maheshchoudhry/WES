# Knowledge Governance

**Version:** v1.0 &nbsp;|&nbsp; **Status:** Active &nbsp;|&nbsp; **Owner:** WES

How organizational knowledge is governed at WES.

| Area | Rule |
|------|------|
| **Ownership** | The Technical Writer owns the Knowledge Base (under the Project Manager); each entry has a named owner. |
| **Review Responsibility** | The relevant domain role reviews entries for accuracy (e.g., Software Architect for engineering knowledge). |
| **Approval Authority** | Domain knowledge is approved by the reporting role; company-level knowledge by the Studio Director. |
| **Version Management** | Entries use semantic versions (v1.0, v1.1, …) with status Draft → Approved. |
| **Change History** | All changes are tracked through Git history in the repository. |
