# Employee Operating System (EOS)

**Version:** v1.0 &nbsp;|&nbsp; **Status:** Active &nbsp;|&nbsp; **Owner:** WES

The Employee Operating System defines **how every AI employee works** throughout WORLD Engineering Studio (WES). It is an operational framework only — not software, prompts, or automation. It builds on the roles in [Volume 03](../../Blueprint/Volume-03-Roles/README.md) and the AI operating framework in [Volume 05](../../Blueprint/Volume-05-AI-System/README.md).

## Components

| Component | Purpose |
|-----------|---------|
| [Employee Lifecycle](./Employee-Lifecycle.md) | Stages an employee moves through, from recruitment to retirement. |
| [Employee States](./Employee-States.md) | Standard operational states during work. |
| [Operational Rules](./Operational-Rules.md) | Ownership, reporting, collaboration, approval, and accountability rules. |
| [Workforce Register](../Workforce-Register.md) | Central register of all employees and their live status. |

## How It Fits Together

Each employee has a **profile** ([`/Employees`](../../Employees/README.md)) that now carries an operational block: status, assignment, skills, availability, capacity, reporting frequency, approval authority, and operational state. The EOS defines the shared rules those fields follow, so every employee operates consistently.
